# Local Supabase Bootstrap

This directory holds the committed local-Supabase testing artifacts for sync-enabled durability work.

Expected files:

- `public_schema_from_dev.sql`
  - Public schema bootstrap snapshot pulled from remote `spread-dev`
  - Used by `./scripts/local-supabase.sh reset`
  - Not generated automatically from SwiftData models; it must be refreshed from the hosted dev schema when the remote schema changes

- `test.env`
  - Generated by `./scripts/local-supabase.sh provision-users`
  - Contains local URL/key values and deterministic test-user credentials
  - Treated as generated/local state and not required to be committed

Refresh workflow:

1. Ensure Docker Desktop is running.
2. Provide `SUPABASE_DB_PASSWORD_DEV` either in your shell or in `supabase/.env.local`.
3. Run `./scripts/local-supabase.sh bootstrap-schema-from-dev`.
4. Commit the updated `public_schema_from_dev.sql` when the shared schema changes intentionally.
